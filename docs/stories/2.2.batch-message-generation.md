## Story
**As a** usuário,
**I want** gerar mensagens para múltiplos leads de uma vez,
**so that** eu possa processar lotes de prospects sem precisar clicar um por um.

## Acceptance Criteria
1. Endpoint `POST /api/leads/generate-messages` aceita body `{ ids?: string[], all?: boolean, force?: boolean }`
2. Com `all: true`, processa todos os leads sem mensagem no JSON
3. Geração é feita sequencialmente (não paralela) para evitar rate limit do Gemini (15 RPM)
4. Retorna relatório `{ processed: number, errors: number, details: Array<{ leadId, success, error? }> }`
5. Leads já com mensagem são ignorados a menos que `force: true` seja enviado

## Dev Notes

### Route Handler — Implementação
[Source: docs/architecture.md#backend-architecture]
```typescript
// src/app/api/leads/generate-messages/route.ts
import { NextRequest, NextResponse } from "next/server";
import { leadsRepository } from "@/lib/leads-repository";
import { generateMessage } from "@/lib/gemini-service";
import type { BatchGenerateRequest, BatchGenerateResult } from "@/types/messages";

export async function POST(request: NextRequest) {
  try {
    const body: BatchGenerateRequest = await request.json();

    if (!body.ids && !body.all) {
      return NextResponse.json(
        { error: "Forneça 'ids' ou 'all: true'" },
        { status: 400 }
      );
    }

    let leads = body.all
      ? await leadsRepository.getAll()
      : await Promise.all((body.ids ?? []).map(id => leadsRepository.getById(id)))
          .then(arr => arr.filter(Boolean) as Awaited<ReturnType<typeof leadsRepository.getById>>[]);

    // Filtrar leads sem mensagem (exceto se force: true)
    if (!body.force) {
      leads = leads.filter(l => l!.message === null);
    }

    const result: BatchGenerateResult = { processed: 0, errors: 0, details: [] };

    // SEQUENCIAL — respeita rate limit do Gemini (15 RPM)
    for (const lead of leads) {
      if (!lead) continue;
      try {
        const message = await generateMessage(lead);
        await leadsRepository.update(lead.id, { message });
        result.processed++;
        result.details.push({ leadId: lead.id, success: true });
      } catch (error) {
        result.errors++;
        result.details.push({
          leadId: lead.id,
          success: false,
          error: error instanceof Error ? error.message : "Erro desconhecido",
        });
      }
    }

    return NextResponse.json(result);
  } catch (error) {
    console.error("[POST /api/leads/generate-messages]", error);
    return NextResponse.json({ error: "Falha na geração em lote" }, { status: 500 });
  }
}
```

### Rate Limit Gemini
[Source: docs/architecture.md#google-gemini-api]
- Tier gratuito: **15 RPM** (requests per minute)
- Processamento sequencial (`for...of`) garante no máximo 1 req/vez
- Para lotes grandes (> 15), considerar delay entre requests em versão futura

### Regras Críticas
[Source: docs/architecture.md#coding-standards]
- **NUNCA usar `Promise.all`** para geração em lote — viola rate limit do Gemini
- Usar `for...of` loop para processamento sequencial garantido
- Relatório `details[]` deve ter um entry para cada lead processado (sucesso ou erro)
