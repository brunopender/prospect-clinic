## Story
**As a** usuário,
**I want** clicar em um lead e ver seus detalhes completos junto com a mensagem gerada,
**so that** eu possa copiar a mensagem e usar na abordagem ao prospect.

## Acceptance Criteria
1. Clique na linha abre painel lateral (Sheet do shadcn/ui) com dados completos do lead
2. Mensagem personalizada exibida em campo de texto copiável com botão "Copiar"
3. Botão "Re-gerar mensagem" chama `POST /api/leads/[id]/generate-message` e atualiza o painel
4. Se o lead não tem mensagem, exibe botão "Gerar mensagem agora"
5. Link direto para o perfil do lead (Instagram/LinkedIn) abre em nova aba

### Componente LeadSheet
[Source: docs/architecture.md#frontend-architecture]
```typescript
// src/components/leads/LeadSheet.tsx
"use client";

import { Sheet, SheetContent, SheetHeader, SheetTitle } from "@/components/ui/sheet";
import { Button } from "@/components/ui/button";
import { Textarea } from "@/components/ui/textarea";
import { useGenerateMessage } from "@/hooks/useGenerateMessage";
import type { Lead } from "@/types/lead";

interface LeadSheetProps {
  lead: Lead | null;
  onClose: () => void;
  onMessageGenerated: (msg: string) => void;
}

export function LeadSheet({ lead, onClose, onMessageGenerated }: LeadSheetProps) {
  const { generateMessage, loading, error } = useGenerateMessage();

  if (!lead) return null;

  const handleGenerate = async () => {
    const message = await generateMessage(lead.id);
    onMessageGenerated(message);
  };

  const handleCopy = async () => {
    if (lead.message) {
      await navigator.clipboard.writeText(lead.message);
      // Toast: "Mensagem copiada!"
    }
  };

  return (
    <Sheet open={!!lead} onOpenChange={onClose}>
      <SheetContent>
        <SheetHeader>
          <SheetTitle>{lead.name}</SheetTitle>
        </SheetHeader>
        <div className="space-y-4">
          <a
            href={lead.profileUrl}
            target="_blank"
            rel="noopener noreferrer"
            className="text-blue-600 hover:underline"
          >
            Ver perfil no {lead.platform}
          </a>
          <div>
            <p className="text-sm font-semibold">Bio</p>
            <p className="text-sm">{lead.bio}</p>
          </div>
          <div>
            <p className="text-sm font-semibold">Seguidores</p>
            <p className="text-sm">{lead.followersCount.toLocaleString()}</p>
          </div>

          {lead.message ? (
            <>
              <Textarea
                value={lead.message}
                readOnly
                className="h-32"
              />
              <Button onClick={handleCopy}>Copiar</Button>
            </>
          ) : (
            <Button onClick={handleGenerate} disabled={loading}>
              Gerar mensagem
            </Button>
          )}

          <Button
            variant="outline"
            onClick={handleGenerate}
            disabled={loading}
          >
            {loading ? "Gerando..." : "Re-gerar"}
          </Button>

          {error && <p className="text-red-600 text-sm">{error}</p>}
        </div>
      </SheetContent>
    </Sheet>
  );
}
```

### Hook useGenerateMessage
[Source: docs/architecture.md#frontend-services-layer]
```typescript
// src/hooks/useGenerateMessage.ts
import { useState } from "react";

export function useGenerateMessage() {
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const generateMessage = async (leadId: string): Promise<string> => {
    setLoading(true);
    setError(null);
    try {
      const res = await fetch(`/api/leads/${leadId}/generate-message`, {
        method: "POST",
      });
      if (!res.ok) {
        const { error } = await res.json();
        setError(error ?? "Falha ao gerar");
        return "";
      }
      const { message } = await res.json();
      return message;
    } catch {
      setError("Falha de conexão");
      return "";
    } finally {
      setLoading(false);
    }
  };

  return { generateMessage, loading, error };
}
```

### Regras Críticas
[Source: docs/architecture.md#coding-standards]
- Link para perfil sempre abre em nova aba: `target="_blank" rel="noopener noreferrer"`
- Copy via `navigator.clipboard` — não funciona em HTTP inseguro (apenas HTTPS/localhost)
